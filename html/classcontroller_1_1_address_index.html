Detailed Description

Manage address indices for user accounts.

Author
Dario Rekowski
Date
2020-02-06

Every user public key (account, address) gets an address index (uint32).
The block.index files contain this indices instead of the full public key (32 Byte)
to save storage space (3/4 saved) and more important memory and for faster look up.
It is number counting up which every new user public key, starting with 1.
Max user public keys per group: 4,294,967,295.
Using folder with first byte of public key and filename with second byte of public key to keeping index file small. File Path starting with first byte from account public key as folder and second byte as file name.
Example: Path for public key: 94937427d885fe93e22a76a6c839ebc4fdf4e5056012ee088cdebb89a24f778c
./94/93.index
